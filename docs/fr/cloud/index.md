Tale Cloud est la version managée de la plateforme. Nous exploitons l'infrastructure, appliquons les mises à jour et les sauvegardes, et hébergeons tes données en Suisse ou dans l'UE — tu te concentres sur le produit. Cloud tourne sur la même base de code que l'édition auto-hébergée : chaque fonctionnalité décrite dans [Platform](/fr/platform) est disponible dès le premier jour.

Choisis Cloud si tu veux profiter de Tale sans exploiter toi-même la stack, ou si la résidence régionale des données et la conformité (ISO 27001, SOC 2 Type II, RGPD) sont obligatoires. Choisis [Self-hosted](/fr/self-hosted) si la plateforme doit tourner dans ton VPC, sur du matériel que tu contrôles, ou dans une build personnalisée.

## Cloud et Self-hosted en bref

| Dimension            | Cloud                                               | Self-hosted                                                                   |
| -------------------- | --------------------------------------------------- | ----------------------------------------------------------------------------- |
| Exploitation         | Tale (nous)                                         | Ton équipe                                                                    |
| Hébergement          | Suisse ou UE (fixé par tenant)                      | Ton infrastructure, où tu veux                                                |
| Mises à jour         | Automatiques, sans interruption de service          | Tu lances `tale deploy`                                                       |
| Parité fonctionnelle | Identique à Self-hosted                             | Identique à Cloud                                                             |
| Adapté à             | Équipes qui veulent Tale sans la charge d'exploit.  | Équipes ayant des exigences de souveraineté, d'air-gap ou de build sur mesure |

## Dans cette section

Les chapitres Cloud couvrent ce qui change quand Tale est exploité pour toi — onboarding, facturation, résidence régionale des données, posture de conformité et les actions d'administration qui n'existent qu'en Cloud (SSO hébergé, domaines personnalisés, export des journaux d'audit).

Chaque documentation de fonctionnalité — chat, base de connaissances, agents, automatisations, intégrations — vit dans [Platform](/fr/platform) et s'applique de manière identique en Cloud et en Self-hosted. Une fois ton onboarding terminé, c'est là que tu trouveras l'essentiel.
