title: Branding
description: Customise the platform name, logo, favicon, and brand colours for your organisation.
---

Branding lets admins customise how Tale looks for everyone in the organisation. Changes apply to the browser tab, the navigation bar, emails, and notification headers.

Available options in **Settings > Branding**:

| Option               | Description                                                        |
| -------------------- | ------------------------------------------------------------------ |
| **Application name** | Replaces "Tale" in the browser tab and header.                     |
| **Logo**             | Image shown in the navigation bar. Replaces the default Tale logo. |
| **Favicon**          | Icon in the browser tab. Separate images for light and dark mode.  |
| **Brand colour**     | Primary colour used for buttons and active states.                 |
| **Accent colour**    | Secondary colour used for highlights and badges.                   |

Changes are applied immediately for every user in the organisation — no reload is required on other clients.

## Logo and favicon

- Upload PNG, JPEG, or SVG files for the logo. SVG is recommended for crisp rendering at all sizes.
- Favicons should be square PNGs of at least 512×512 px. Tale resizes them for the different browser sizes automatically.
- Light-mode and dark-mode variants let you use a dark-on-white logo for light mode and a white-on-dark logo for dark mode.

## Colours

Colours are entered as hex codes. The platform enforces a 4.5:1 contrast ratio against the background — if a chosen colour falls below it, an accessibility warning appears and the colour picker suggests a close alternative that passes contrast.

## Resetting

Click the reset icon next to any field to revert it to the Tale default. Resetting the application name and logo together restores the default Tale branding.
